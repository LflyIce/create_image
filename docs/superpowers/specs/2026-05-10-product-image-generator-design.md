# 产品图片生成器设计文档

## 目标

创建一个用于生成产品图片的网站。用户上传一张示例产品图，例如门帘或蟾皮材质参考图，再输入图片生成提示词。系统生成两类结果：中间列展示根据提示词生成的风格图，右侧列展示把风格图贴到示例产品后的产品贴膜图。

第一版是完整前端原型，使用前端点数扣减和可替换的生成模块。项目已接入 Doubao Seedream 4.0 文生图接口：浏览器请求本地 `/api/doubao/images`，再由 Vite 开发服务器读取服务端环境变量 `ARK_API_KEY` 转发到火山 Ark，避免把密钥暴露到浏览器包里。

## 用户体验

页面采用三列工作台布局：

- 左侧列：上传示例图片、填写提示词、选择图片规格、选择生成数量并开始生成。
- 中间列：展示根据提示词生成的风格图片。
- 右侧列：展示把中间风格图套用到示例产品后的贴膜效果图。

每生成一张图片消耗 1 点。选择 4 组时，会生成 4 张风格图和 4 张产品图，共消耗 8 点。界面需要展示当前点数、预计消耗、缺少输入或点数不足时的禁用状态，以及生成中的加载态和生成后的结果卡片。

## 视觉方向

页面采用“工业产品影像工作室”的视觉方向：

- 深色精密工作台背景。
- 琥珀色作为主要操作色，用于开始生成和关键状态。
- 薄边框、灯箱式结果区和网格纹理强化生产工具感。
- 使用 Ant Design 组件作为基础交互组件，并通过 CSS 变量统一成项目自己的视觉系统。

## 范围

本版本包含：

- Vite + React + TypeScript 项目。
- Ant Design 表单、按钮、卡片、统计、标签、空状态和加载组件。
- 三列响应式工作台布局。
- 本地图片上传预览。
- 提示词、图片规格和生成数量控制。
- 模拟图片生成和延迟加载状态。
- Doubao Seedream 4.0 图片生成接口代理。
- 浏览器内生成的确定性 SVG 占位图片。
- 本地点数余额和按张扣点。
- 空状态、加载状态、成功状态和错误状态。

本版本不包含：

- 真实登录系统。
- 真实充值或支付流程。
- 生产环境 API 服务部署。
- 服务端图片合成。

## 架构

`src/App.tsx` 负责页面状态、三列工作台和 Ant Design 组件组合；`src/lib/doubao.ts` 负责构造 Doubao Seedream 4.0 请求并通过本地代理生成图片；`src/lib/generator.ts` 保留本地模拟生成兜底。开发期代理写在 `vite.config.ts`，需要在启动开发服务器前配置 `ARK_API_KEY`。

当前接入基于用户提供的文生图示例。由于该示例没有包含图生图或参考图参数，右侧“产品贴膜图”会通过产品展示提示词生成，而不是把上传示例图真实传入模型做图像编辑。

## 测试

使用 Vitest 测试生成成本和生成结果结构。使用 React Testing Library 测试主工作台行为：未上传或未填写提示词时禁用生成按钮、生成数量改变后预计点数更新、上传示例图后可以生成左右两列结果并扣减点数。
